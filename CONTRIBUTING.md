# 贡献指南

## 先讨论边界

提交前先确认问题属于哪个现有 skill（智能体技能）。本仓库最重要的质量标准是职责边界：如果一个改动让两个 skill 都能回答同一类请求，应先在 issue（问题单）中说明为什么不复用或调整既有 skill。

新增 skill 请说明：

- 解决的具体动作与目标使用者；
- 可触发它的自然语言表达；
- 输入、输出与可验证结果；
- 明确不做什么，以及与相邻 skill 的区别；
- 所需工具、环境变量或外部服务。

## 提交要求

1. 阅读根 `README.md`、`AGENTS.md` 和目标目录的说明。
2. 修改 `SKILL.md` 后，同步检查上级 `README.md` 中的流程图、目录表与边界描述。
3. 脚本改动运行对应最小测试；文档改动检查链接和路径。
4. 不提交 API key（接口密钥）、Cookie（会话凭据）、个人/客户资料、私人路径或未获授权的素材。
5. 使用 `Conventional Commits`（约定式提交），如 `docs: 补充安装说明`。

## 风格

- 使用具体、可执行的语言，少写抽象口号。
- 以真实失败场景或验证假设支撑优化，不为“更完整”扩大规则。
- 保留中文为主的写作风格；技术术语可附英文原词。
- 面向公开使用者时，把私人上下文改为可理解的前提或可替换占位符。
