# 安全政策

## 报告安全问题

请不要在公开 issue（问题单）中披露以下内容：

- 已暴露或疑似暴露的 API key（接口密钥）、token（令牌）、Cookie（会话凭据）或账号数据；
- 可导致敏感素材、个人 vault（知识库）或本地文件被意外上传/读取的漏洞；
- 外部服务调用中可能泄露输入内容的问题。

请通过 GitHub（代码托管平台）账户的私信或仓库所有者公开提供的私下渠道报告，并附上最小复现步骤、影响范围与脱敏后的证据。收到报告后会确认、评估影响并在修复后公开说明。

## 使用者责任

- 在运行带外部 API（接口）或上传能力的脚本前，确认你有权处理相应素材。
- 只通过环境变量提供密钥；不要把 `.env`（环境变量文件）、shell（命令行）配置或终端输出提交进仓库。
- Fork（派生）后请自行复查新增的脚本、依赖和自动化步骤。
