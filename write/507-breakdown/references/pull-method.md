# 507 Video Pull 拉片方法

## 目标

拉片不是普通总结，而是为了**借鉴别人怎么把视频做出来**。

第一版重点回答三件事：

1. 这个视频**讲了什么**。
2. 这个视频**怎么组织信息**。
3. 这个视频**用什么画面、字幕、剪辑手法把信息送达**。

## 三层分析

### 1. 叙事与结构

- 开场 hook（钩子）是什么
- 信息怎么推进
- 论点 / 节拍怎么切换
- 结尾怎么收束

### 2. 视听语言

- 镜头（景别 / 机位 / 运镜）
- 剪辑（切换频率 / 转场 / 叠加）
- 色彩与光影
- 字幕与 UI 呈现

### 3. 可借鉴点

- 哪些手法值得借
- 哪些手法只适合原视频，不适合泛化
- 哪些手法后续可被 `507-video-remake` 组合

## 两层分段

### 分析层

先按小窗处理证据，避免 agent 输入过载：
- 单窗时长：约 6-8 秒
- 单窗台词：约 2-3 个字幕 / ASR 块
- 单窗画面：约 6-8 个关键帧引用

### 输出层

基于小窗结果按**语义**合并成最终段落：
- 不按固定 10 秒硬切
- 一个完整观点可以横跨十几秒甚至更长
- 只在信息功能变化时切段

## 文字证据链

优先顺序：
1. 平台原生字幕
2. 平台自动字幕
3. 本地 ASR
4. 帧内 OCR

一旦前一层成功，就不要继续往下混用。
