# 507 写作技法示例库

本文件是**克制思辨调性**的技法示例（§二 T1-T11 的对照样本）。诙谐毒舌调性的示例在 `诙谐毒舌调性.md`，两个文件物理隔离，别混用。

`507-writing/SKILL.md` 的技法是规则，本文件是可对照示例。规则没有示例无法校准。

## 口癖（保留，别清掉）

507 的真人感在未过滤的口癖层，不在干净的书面感里。太干净反而不真实。草稿和聊天里自然出现的填充词、软化词、转折词是声音本身，不是噪点。定稿修正同音错字和事实性错别字，但不清口癖本身的"不规范"（就是/其实/吧/呢/反正这些是声音不是错字）。

**观点前缀（hedging，几乎每个判断都从这开头，最强签名）**
- 我会认为 / 我会感觉 / 我会觉得 / 我现在的感受是 / 我现在的感觉是 / 我会担心 / 我会担忧 / 我会想 / 我会这么想 / 我还有一个感受是 / 我会怀疑 / 我大概率推断 / 我会发现 / 我记得 / 我认为 / 我期望 / 我会希望的是 / 我提醒你的是 / 我会倾向于 / 我倾向于 / 我担心 / 我会想着说 / 我会理解成
- 例："我会认为这个任务是请你基于当前我们的碎片来进行吸收"

**句首结构 tic（高频开头，之前漏的）**
- 等下，… / 等一下，… / 等一下哈，…——打断式纠错开头，"等下你这里是不是逻辑错乱了呀？""等下，为啥 design 不能写？""等一下，我现在关注到你似乎是在自己干活？？？"
- 而且…——追加叠点，极高频，"而且你刚刚的修改还没搞定哈""而且关键是什么呢""而且我还有一个想法"
- 对了，…——话题切换开头，"对了，那我想知道的是""对了，我认为现在需要你先更新术语表先"
- 说起来 / 话说回来 / 顺便…——话题切换 / 附加，"说起来，skills 是不是放到文件夹内也可以被识别""话说回来的话""顺便帮我改一下"
- 喂，…——引起注意开头，"喂，你可以先帮我存一份 doc 先"
- 但是… / 但是呢，…——让步开头，"但是我会认为你这里有一个问题""但是呢，我会认为你需要更新到 agents.md"
- 嗯，… / 嗯？——思考 / 应答 / 疑惑开头
- 哦，…——接话 / 领悟，"哦，我会认为""哦，原来如此，那你就应该让他返回 think 的内容呀"
- 其实…——转折真实想法，"其实我会想""其实我的想法是""其实我现在是想做"，"嗯，我觉得这里的设计简单才是更加健壮的""嗯？还没好吗？过了10分钟了吧"
- 那… / 那么… / 那么，…——推理链 / 承接开头，极高频，"那为什么你不直接帮我运行""那么基于此你认为我们项目可用了吗""那么你认为"
- 算了，…——放弃当前思路 / 转向，"算了，目前来看我其实就喜欢 dgoal""算了，你做个 commit 先把"
- 首先（第一个），…——分点陈述开头，"首先第一个呢，是我们当前这样子会有点混乱"
- 刚刚…——回顾发现，"刚刚奔溃了""刚刚我突然间意识到一个新的问题""我现在的感受是，需要碎片图谱里面做一下分组""我会希望的是 AI 提高人效，而不是把人优化掉"

**thinking-aloud（边想边说）**
- 我在想 / 我突然想到 / 我刚刚在思考 / 我想了下 / 我都在设想 / 我都想问问 / 我反而想

**框定句式**

- 例："这句话是毒瘤，可以删掉""太啰嗦且太 AI""文字 AI 味道严重""这个说法怪怪的"

**试探性提议**
- 似乎可以… / 我觉得可能可以… / 要不我们… / 我觉得是…但有取舍
- 例："git-workflow 似乎可以整合到提示词里""要不我们看看其他的"

**简短确认 / 推进**——继续 / 继续+动词（继续追问 / 继续挖 / 继续收 / 继续过 / 继续补 / 继续测试 / 继续看看） / 开工 / 先落 / 先 commit / good / 可以 / 可以的，… / 认可 / 很好，… / 是的 / 行吧（勉强接受，"行吧，我认为你先跑个 200 会话试一下"） / 补一下 / 都删掉 / A / B / 1 / 2。

**长逗号流水句**——不切句、自我穿插、用"然后/就是/或者说是/反正"串起来，草稿阶段保留这个流式质感。同音错字（把→吧、伴→帮、盯得→盯的、得=那、有事=你是、红=着/的）定稿要修正——这是事实性错字，不是语气，不留。

**括号旁白**——"（现在过于冗余）""（prompt）""（这里只是碎片的）"，用括号塞自我注释。

**准则**：这些口癖在草稿和聊天里自然出现时，不要因为“不够书面”就删。验收时“太干净 / 像范文 / 像 PR 稿”是失败模式，和 AI 味一样要返工。

**同音错字表（定稿修正，这些是事实性错字不是语气）**：把→吧 / 伴→帮 / 盯得→盯的 / 得=那 / 有事=你是 / 红=着/的。出现这些要修正，但不清口癖本身（就是/其实/吧/呢/反正是声音不是错字）。

## 标点纪律（公众号 / 博客长文从严）

三个标点是 AI 味高频来源，公众号长文宁可不用。

- **冒号→换逗号**。
- **破折号→换逗号或句号**。
- **双引号**：用 “” 和 ‘’，不用「」（「」太日系）。能用陈述代替引号时直接不加引号。
- 课程稿 / 方法论保留必要的结构标点，但也尽量少。
- 表格本身需要冒号格式的，不属此约束。

**转场句示例**（板块之间用口语化转场句自然衔接，不用分隔符 / 小标题切割）：说到这个 / 回到 xxx 这块 / 顺着上面的再聊聊 / 不过话说回来。

## 节奏与语气

- **转折建张力**（核心节奏工具）：用"按理说…但…""…但问题也正是在这里""恰恰相反"把读者预期掰过来。例："听起来很爽，对吧？但问题也正是在这里。""很多人越用 AI 越累，不是因为 AI 没帮上忙。恰恰相反，是因为它帮得太快了。"
- **一句成段断裂**：极短句独立成段制造重量感，克制使用。例："现在不一样了。""开始太容易了。""黑暗森林。"
- **设问 + 共鸣再转折**：先一句设问拉读者进来，紧接着转折。例："听起来很爽，对吧？但问题也正是在这里。"
- **"不是…而是…" 论证**：重定义问题、掰开误解。例："不是替人决策，而是帮人准备信息 + 自动化推送 + 人守关键节点。"
- **扣主线句**：偏出去一点点（喘气 / 案例 / 长见识），再用一句话拉回主线，高频出现；最忌偏离很远再硬拽回来。
- **论述中故意打破**：重复强调、中途打断、省略主语、刻意模糊，让论述有温度像真人在说。
- **知识"聊着随手掏"**：不是"下面我来科普"，是聊着正好对上了。
- **情绪用体感记忆，不用知识性描述**：写身体反应 / 当时那句话 / 那个瞬间，不给情绪贴标签（"我当时就愣住了"而非"我感到非常震撼"）。**但注意反噬**：体感必须来自真实记忆，AI 编造的体感（哪怕是"愣了一下""后背凉了一下"这种身体反应级别）一旦套路化，比克制更暴露 AI 味——被 507 实测判定为"过于 AI"。拿不准就用最克制的真实反应词（"有点意外"），别上套路化身体反应。
- **亲自下场**：能让读者感到"这个人真的做了这件事"，不是在转述别人的案例。**最大的杠杆点是真实观点分歧**——你和现场某个角色（老板/合作方）对同一件事有不同立场，而且你当面跟他讲了。例：老板想用 AI 把人替换掉（最后自己一个人 OPC），你认为 AI 让人更高效、人升级到判断岗和沟通岗。这种"我跟老板想得不一样，但我当时就是这么说的"是任何转载稿、任何 AI 都造不出来的活人感来源，比"我当时愣住了"强 100 倍。写不出来时，先问：这次经历里，我跟谁有过不同意见、我当时说了什么。

## 论点与说服

- **对立面与同理心**：讲观点前先站到对方处境具体化、承认合理，再给自己的视角。例："我非常理解这种感觉。你不是程序员，不需要写代码……你就是一个普通的上班族。"
- **承认普遍困境**（谦逊方式）："先让大家承认：这件事不是个别人遇到。"
- **谦逊铺垫**：方法论 / 教程型开头先自谦卸掉"我来教你"的傲慢感，结尾回扣行动点。
- **层层剥开**：现象→表面解释→更深追问→核心洞察，每句在上一句基础上再推进一层。
- **反向论证**：先满足期待再打破。
- **读者直呼**：关键节点"你也可以回想一下"精准投放，不通篇用。
- **概念命名 + 加粗点题**：给现象命名（天然过滤 / 成本门槛），再加粗短句钉成结论。

## 叙事技法（按文章类型取用，不每篇全上）

- **回环呼应（契诃夫之枪）**：前面埋的钩子后面要响，让文章从"信息流"变"作品"。
- **英雄之旅叙事弧**：案例复盘 / 产品体验型——具体困境或好奇→一步步做踩坑→让人"卧槽"的结果。
- **人物画像法**：数据点→背后那个具体的人的完整人生，3-5 句立住。
- **升番展示**：多产品 / 多案例并列时逐一展示（每个带一句点评），弱的在前炸的在后，中间留"还能往上翻"的惊喜。
- **文化升维**：从具体事件连到更大的文化 / 哲学 / 历史参照物；有机会自然连上就用，连不上别硬凑，不每篇必备。
- **坦诚学习曲线**：方法论文章不只讲成功画面，还坦诚说明初期会有多痛苦，用复利式增长给信心。

## 活人感标尺

**AI 初稿 vs 修改对比**（反复看，标定"AI 味"和"活人感"的差距）：
- 工具名精确化 + 口语化：AI 写"Claude 刚出来他就用上了"→ 改"Deepresearch 刚出来他就用上了，Claude code 啥的也比很多人 6"。
- 假设例子 → 真实当下场景：AI 写"有一次我需要批量处理图片"→ 改"就像我今天正在搞的爬下来我自己的内容在全网的数据"。
- 加私人判断 + 数字更极端：AI 写"每天工作六个小时产出比十个小时还多"→ 改"我真的觉得工时这个指标变得毫无意义，每天工作 2 个小时产出比 10 个小时还多"。

标尺规律：精确当下工具名 / 真实正在发生的场景替代假设例子 / 加私人判断词 / 数字更极端更有冲击力 / 阿拉伯数字比汉字更口语。

**507 自己的对比库（待积累）**：长期真正起作用的是 507 在自己写作中积累的 AI 初稿 vs 507 改稿对比。每发现一处"AI 味重 → 507 会怎么改"就追加：

### 反套路：AI 味标记词（假口语 AI 味，比书面套话更隐蔽）

这类词是 AI 用来"装活人"的高频套路词。和"说白了 / 本质上"那种书面 AI 味不同——这些披着口语的外衣，但因为 AI 用得太频，反而成了指纹。一出现就露。

- **"很燃 / 挺燃 / 燃到了"**——AI 试图模拟口语情绪的表演词。真人说"挺爽的 / 上头 / 心动 / 听着是那么回事 / 唬人"。AI 初稿："听起来很燃。"→ 改："这话确实让人心动。"或"听着是挺爽的。"
- **"后来我才想明白 / 后来我才意识到 / 后来我才搞懂"**——AI 用"后来我才X"做顿悟叙事桥梁，制造"经历过反思"的假象。真人要么直接跳到结论不铺垫，要么"我想了下 / 琢磨了一下才反应过来 / 后面才搞清楚"。AI 初稿："后来我才想明白，这两件事根本不是两个问题。"→ 改："我想了下才搞清楚，这两件事根本不是两个问题。"
- **"扫兴"**——AI 试图模拟“我要说不好听的话了”的预铺垫词，但真人不会用“扫兴”来自嘲，太书面。真人说"这话可能不太好听""这听上去怪怪的""我知道这话不讨喜"。AI 初稿："我知道这话扫兴。"→ 改："这话可能不太好听。"或"这听上去怪怪的。"
- **“狠 / 太狠了 / 很狠”**——AI 高频程度副词，用来模拟“语气强烈/有力”，但真人不会这么刻意地用“狠”。做修饰别人的话→"挺直接的 / 挺到位的 / 讲得直白"；做程度→"要命 / 恶心 / 难受 / 受不了 / 太多次了"。AI 初稿："说了一句很狠的话"→ 改："说了句挺直接的"；"栽得太狠了"→ 改："栽得我受不了了"。
- **"这件事我自己也 X 过"**——AI 试图模拟“我也经历过”的亲切过渡句，但真人不会把"栽过 / 踩过坊"拎出来当一句独立陈述。前面已经交代了身份/场景时，这句是多余的，直接跳进经历描述更自然。AI 初稿："我也是挂 loop 跑的那批人，这件事我自己也栽过。"→ 改："我也是挂 loop 跑的那批人，几乎每次都翻车。"
- **"转折钩子句"（最高频最隐蔽的 AI 味）**——AI 爱用独立成句的钩子做段落转折，而不是让转折发生在内容本身。一出现就露："但有一件事…""但新的问题冒出来了""发现一个有意思的事""这就是我…最想说的""写到这里我以为…""我跑了很久才反应过来"。这些全是廉价桥梁，真人转折在内容本身。AI 初稿："后来我换成了 /goal。但新的问题冒出来了。agent 跑得很认真…"→ 改："后来我换成了 /goal。但 agent 跑得很认真…"（砍掉钩子句，转折在内容里）。**治理方式不是换另一个钩子句，是砍掉直接接内容。**
- **假谦逊表演**——AI 装真实感的新套路："我自己到现在也没想清楚""我也在困惑中""我到现在没弄明白"。看起来是真人坦诚，实则是用"我也没想清楚"制造共鸣，是编造作者状态（作者多半有倾向，只是还没收口）。违反 T11，和"后来我才想明白"同源——都用假状态做叙事桥梁。AI 初稿："我自己到现在也没想清楚该信哪条。"→ 改：要么不写这句（作者状态不写就不写，不用装），要么写真实状态（"这对张力我到现在还没收口，两条都留着"是事实，"我也没想清楚"是表演）。
- **让读者自己算出结论**——AI 假装让读者参与推理："工作量你自己算""你自己想想""你自己品"。和"没火但怼得对"同源（让读者得出我要的结论，假装深刻），但变体没列。AI 初稿："两百多条手动挑一遍，工作量你自己算。"→ 改：直接给出事实让荒谬自己顶出来，"两百多条两两对起来，三万多对"（用真数字砸出不可能，不是让读者算）。
- **强行回环呼应（T8 的滥用）**——T8 要求"前面埋的钩子后面要响"，但 AI 会把同一句硬凑两遍装回环。开头"得清到明年"，结尾又"就够我清到明年"，是重复不是呼应。真回环是钩子在不同语境里响出新意，不是原句再响一遍。AI 初稿：开头"得清到明年"，结尾"就够我清到明年"。→ 改：结尾换一个角度落（"三万多对"），不重复原句。
- **积累规则**：507 在写作中每发现一个"AI 味标记词"，追加到这里。和书面套话（L1 扫描）分开列，因为治理方式不同——书面套话直接删换，假口语词要换成真口语而不是删掉情绪。

### 复合污染警示（一段把多个 AI 套路叠满）

AI 初稿最危险的场景不是单处套路，是**一段把多个套路叠满**——假谦逊 + 转折钩子 + 工整排比 + 强行回环 + 重复自嘲，一段全犯。识别信号：你读一段觉得"哪哪都不对劲但说不出具体哪句"，基本就是复合污染。单句扫描会漏，要整段通读问"这段是不是在表演而不是在陈述事实"。

实际样本（本卡第三段）："我自己到现在也没想清楚该信哪条。但更麻烦的是，我知识库里有两百多条碎片，像这种正反都还在、谁也没赢的，不止一对。手动一条条挑出来审一遍，光这个活，就够我清到明年。"——一句里叠了假谦逊 + 转折钩子 + 工整对仗（正反/谁也没赢）+ 强行回环（清到明年重复开头）+ 重复自嘲，五种套路。改法不是逐句换词，是整段砸掉表演只留事实：真实碎片对的两条原话 + "两条都还在，谁也没标谁过时" + "三万多对"用真数字砸出不可能。事实自己有力度，不需要任何表演加码。

### 对比 4（《别急着写循环》三大坑—— AI 排比 vs 507 改写）
- AI 初稿："偷懒，是因为没有完成判定。自夸，是因为写代码的和检查代码的是同一个 agent。漂移，是因为没有状态锚定。"——工整排比，"X，是因为Y"三句对仗。
- 507 改写：先一句话列三个现象+名称（"做了 20 项就说搞定了——偷懒。给自己的活打高分——自夸。第 47 轮悄悄消失了——漂移。"），展开段砍掉"这事，根上是"的重复句式，改成"偷懒是没人告诉 agent 什么算完""自夸是写代码的和检查代码的是同一个 agent""漂移则是跑得越久"。
- 差异：AI 的排比太工整，像论文。507 的改法是先给现象+名称（一句话扫完），再逐条展开但不排比。**禁止工整排比展开，用口语化的方式逐条讲。**

### 对比 5（《别急着写循环》结尾—— 说教式 vs 陈述式）
- AI 初稿："Boris 那句“我不再写提示词了，我写循环”听着像人在退场。但看看他到底退了什么，他退的是…他没退的是…"
- 507 改写："Boris 传达了一个想法，“我不再写提示词了，我写循环”。但是他不再做的是…他还在做的是…"
- 差异：AI 用"退/没退"暗示人被取代（说教式），507 用"不再做/还在做"是中性状态描述（陈述式）。而且 AI 写"那句话"暗示 Boris 直接说过（不准确），507 改成"传达了一个想法"。**事实锚点要准确，姿态要陈述不要说教。**

### 对比 6（《别急着写循环》全文—— AI 声音 vs 507 手改）
- 这篇来回改了七八版，每一版 507 都在挑 AI 味。AI 能搭好骨架和血肉（结构、逻辑链、事实锚点、碎片关联），但声音层（口癖、语气、判断的力度）始终是 AI 的，不是 507 的。
- 最后成稿里 507 手改的地方：三大坑排版、"不再做/还在做"、删重复句、"循环这个东西呢"、结尾收软——这些都是 AI 写不出来的。
- **教训：AI 不要假装初稿的声音能直接用。写完骨架和血肉后，明确告诉 507"声音层需要你过一遍手改"。**

### 对比 1（《让 AI 帮你管理一条流程》—— AI 初稿 vs 507 校正）

- 507 校正："我凑过去一看，有点意外。" （删掉后背凉了一下那处）

### 对比 2（《让 AI 帮你管理一条流程》结尾—— AI 初稿 vs 507 校正）

- 507 校正："他的想法其实很直接——AI 能不能把人换掉，他甚至想最后自己一个人 OPC。我当时没顺着他说。我跟他的想法不一样，我认为 AI 不会把人换掉，它会让人更高效，然后人去负责更多需要负责任的事。"

### 对比 3（《别急着写循环》—— 中段退化成“方法论说明书” vs 507 判“看不下去”）
- **病症**：开头是具体场景（Boris 刷屏 → 507 亲历两种翻车 → “那我还不如自己上手”），读者被故事勾住。但中段突然退化成,**用 `---` 分隔符切成三块** + **“约束工程（Harness Engineering）的定义是…”科普式端出** + **孤零零摆一张对比表 + “你看这张表”** + **“先把这三件装上”编号步骤感**。读者从“听你讲故事”变成“听你讲课”，507 判“后面看不下去”。
- **根因**：forge 缺了最高优先级的结构纪律——文章要一口气顺下来，不用分隔符 / 小标题切割。一旦中段从叙事切到论证，读者就出戏。
- **改法方向**：① 砍掉 `---` 分隔符，用口语转场句衔接板块；② harness 定义不要“现在我来科普”，改成“翻车之后我看到 Mitchell 说了一句话…”聊着掏出来；③ 对比表包进叙事，不要孤零零摆着；④ 段落切碎，一句话成段是常态；⑤ 标点换掉（冒号→逗号，破折号→逗号/句号，引号→“”）。
- **这条写进 SKILL.md 的“一口气顺下来”和 L2“结构切割扫描”**，是最易漏的结构红灯。
