# 507 选题与创意方法

`507-writing` 的 fragments 阶段（挖素材、还在想还在观察）和选题判断会用到这里。来源：卡兹克「内容方法论白皮书 v2.0」，507 化取舍。完整背景见 vault `doc/20-能力参考/01-卡兹克风格创作skill参考.md`。

## 1. 选题交集模型

一个好选题 = **你的专业领域 × 读者的普遍兴趣 × 当下的时间节点**。

- 光是专业 → 曲高和寡。
- 光是读者兴趣 → 流于肤浅。
- 光是蹭热点 → 昙花一现。
- 三者交集才是最精准的引爆点。

507 的"专业领域"= AI 协作 / 工程方法论 / 思考方法；选题要同时回答：这是 507 真信且专业的判断吗？读者普遍会关心吗？现在这个时间点写为什么？

## 2. HKR 质检（动笔前每个选题必过）

- **H（Happy）**：足够有趣、有悬念？标题和开头能让人好奇想点开？
- **K（Knowledge）**：有信息量？看完能学到新东西 / 多一个判断？
- **R（Resonance）**：能戳中情绪？让人"对对对我也这么想"？

S 级三项兼备，及格至少两项，只占一项重新审视方向，零项坚决放弃。

## 3. 角色代入法（发散审视选题）

审视选题时代入不同角色：
- **一个很忙的普通用户**：能帮我省时间 / 解决具体麻烦吗？
- **一个爱玩的朋友**：够有趣吗？能让我产生"卧槽还能这样？"的惊叹？
- **一个焦虑的学习者**：能让我学到新东西、有新角度吗？

三个角色都打不动，选题就悬。

## 4. 选题来源（按 507 的场域调）

卡兹克的来源是 X / Reddit / 小红书 / 即刻 / 微博抖音 B 站 / 读者社群。507 的场域不同，对应替换成自己的信息源：技术社区 / 读者反馈 / 自己踩的坑 / 碎片图谱里聚拢到能成选题的簇。**核心机制是"多源交叉 + 至少两三个渠道同时火才算真破圈"**，来源列表按 507 自己的场域维护。

## 5. 创意案例工作法（案例型文章用）

### 三类锚点

- **热点锚点**：当下最火的电影 / 电视剧 / 社会事件 / 网络热梗——流量最大公约数。
- **痛点锚点**：用户最高频、最头疼的问题——实用价值。
- **炫技点锚点**：AI 工具本身最颠覆的核心功能——技术惊叹。

好案例往往同时命中两类锚点。

### 包装成微型故事（四步）

1. 亮出**挑战**：想解决什么问题？
2. 展示**脑洞**：想到了什么骚操作？
3. 秀出**过程**：怎么一步步实现的？（截图、核心 Prompt）
4. 引爆**结果**：最终那个让人"卧槽"的作品。

### 准则

案例必须有独立的观赏价值、创意价值和启发价值。**如果自己都 High 不起来，坚决不发。**

## 6. 内容节奏

节奏不是文笔问题，是一套让读者一路往下读的推进系统。好的节奏像波动：每次围绕主线偏出去一点点（喘气 / 长见识 / 看案例），再用一句话拉回来继续推进。

- **扣主线句**要高频出现，不需要长，一句就够，但频率要高。
- **最忌讳**：突然偏离主线很远再硬拽回来，读者要花脑力"顺逻辑"，心流断掉。
- **节奏优先级**：专业（底子）→ 观点（方向）→ 主线（绳子）→ 节奏（牵引力）。专业和观点没立住，光靠节奏是空转。
- **对标参考**（卡兹克的）：影视飓风、何同学、小狮日记、毕导、小 Lin、九边。507 按自己的取向另选对标。

## 7. 文章原型与写法重心（写前先判型）

- **调查实验型**：亲自下场做一件事边做边报道。重心在过程叙事 + 层层发现 + 每步带真实反应。
- **产品体验型**：拿到产品带读者一起玩。重心在场景驱动 + 真实感受 + 跟其他产品的自然对比。
- **现象解读型**：观察一个现象层层深入。重心在好奇心驱动 → 研究 → 知识掉落 → 哲学升维。
- **工具分享型**：分享工具 / Prompt 但用个人故事包裹。重心在个人故事铺垫 → 自然引出工具 → 自己的使用过程 → 效果展示。
- **方法论分享型**：系统性分享长期经验。重心在每节落到可执行行动 + 坦诚学习曲线 + 谦逊铺垫 + 结尾回扣行动点。
