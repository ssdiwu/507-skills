# 锻造（forge）

把创意和创作材料共创成**可发布的书面成品**。默认不是一次生成，也不是逐段审批，而是先确定当前写作合同，由 Agent 写出完整版本，再吸收用户反馈统一重写，直到全文成立。

## 一、职责与边界

**负责**：长文成文、已有草稿重组、同一创意的书面平台保真改编。

**不负责**：
- 无边界挖掘素材或重新发明选题（归 `507-mine` / `507-fuse`）；
- 讲稿、PPT、课程口播（归 `507-stage`）；
- 正式方案或计划（归 `507-frame`）；
- 封面、配图、HTML、分页视觉和最终渲染（交给设计或图片 skill）。

`shape / beats / edit` 只是 Agent 内部可取用的写法，不向用户暴露为模式。文章也没有全局“调性”开关：事实处可以克制，荒诞处可以毒舌，表达力度随内容变化，但作者立场与中心判断必须一致。

## 二、自动入口

1. 先读项目已有的 `AGENTS.md`、根 `README.md`、`doc/README.md`、术语表，以及创意区／作品区约定。
2. 查找当前创意卡和已有主稿；项目已有路径约定时直接使用，不再询问保存位置。
3. 已有经用户确认的创意卡，先检查其是否已有按核心争点整理的材料；齐全才把它作为当前依据。
4. 没有创意卡但中心判断和材料已明确，先自动建立或补齐创意区；若原始材料尚未碎片化进入 `507-mine`，若尚未比较竞争解释进入 `507-fuse`，再返回成文。
5. 外部事实缺口进入 `507-research`，写作碎片缺口进入 `507-mine`，竞争解释进入 `507-fuse`。存在真实竞争路径时，携带候选进入 `507-grill` 让用户决定中心判断，再返回 forge；材料只支持一条路径时，说明收敛理由后可继续。不让用户手动调度技能链。

不询问内部写作模式；需要用户选择中心判断、读者承诺或风险边界时统一进入 `507-grill`，不在 forge 内临时发明一套确认流程。

## 三、写作合同

动笔前确保创意区存在当前有效的：

- **中心判断**：全文最终要让什么判断成立；
- **读者入口与承诺**：读者正处在哪个具体困境、决策、误解或未解问题中，因此需要这个判断；若文中承诺提供可复制资源（提示词、模板等），写清它在哪里出现、读者能拿走什么；
- **论证链**：读者按什么顺序被带到这个判断；
- **核心争点材料**：每个关键推断的支持证据、反例、替代解释、适用边界/缺口与时间或制度语境；
- **证据与边界**：哪些事实承重，哪些只是作者判断；个案能证明什么、不能推出什么；哪些材料会动摇主张。

读者入口只负责建立相关性和阅读预期，**不承重事实**：不得为了代入感编造故事、堆情绪词，或让个案替代证据。

若核心争点材料缺失，写作合同未成立：按缺口分别进入 `507-research`、`507-mine` 或 `507-fuse`；需要用户选择候选路径时进入 `507-grill`，不得直接起草。只有材料经核对后只支持一条路径，才能带着收敛理由进入成文。

“锁定”指当前版本可追踪，不代表不能改：
- 用户明确给出新中心或新顺序时，简短复述变化后直接更新创意区并统一重写；
- 反馈看似局部但可能改变全文时，携带新的写作合同进入 `507-grill` 确认用户取舍，再返回统一重写；
- 禁止把改变主线的新判断局部塞进旧稿。

## 四、共创循环

1. 读取完整创意材料与已有主稿；先确定项目约定的唯一正文真相源，平台稿只能从它派生而不能成为并列主稿；确认写作合同中的核心争点材料完整后，先形成一篇可从头通读的版本，不逐段等待确认。
2. 用户反馈若只影响局部，精确修改；若改变中心、论证链或某个核心争点的解释，按写作合同规则处理受影响全文；新出现的外部事实缺口进入 research，材料缺口进入 mine，竞争解释进入 fuse，不局部硬塞。
3. 每次写入前重读目标主稿，保留已有手改，不另建同题平行稿。
4. 成文中新增的一手来源、二手参考、案例、反例、替代解释、事实边界和新判断，先回写创意区，再用于作品。
5. 创意区保存“为什么写、拿什么写”；作品区只保存具体成品及成品实际使用的附件。
6. 平台改编先继承原写作合同，可改变篇幅、分页和信息密度，不得借适配之名重写成另一篇；视觉实现另行路由。

不要从用户的协作对话里机械复制“我会认为”“理解一下”等口癖。作者声音来自事实选择、因果关系、判断力度和共创中的真实纠正，不是聊天词频模仿。需要个人表达时，不靠第一人称、口癖或情绪词制造“活人感”；只写入会改变判断的具体上下文：亲历过程、可回溯证据、失败、取舍与适用边界。缺少这类材料时，外部事实进入 `507-research`，可复用观点或经验进入 `507-mine`，不用语气模仿补洞。

### 真实链与图片证据

当文章来自作者亲历的尝试、协作或判断变化时，优先按真实因果组织：**未解处境或失败尝试 → 出现的工具/方法 → 可见的产物 → 他人或现实给出的具体反馈 → 作者据此作出的取舍**。工具不是开场的主角，而是对前一处困境的回应。

图片不是配图占位。每张图必须承担一个可说清的作用：证明此前确实卡住、展示产物是什么、或让读者看见反馈如何发生。图片放在它所证明的动作之后、由它引出的反馈之前；不要把“反馈图”放在“产物图”前面，也不要用正文替图片再讲一遍读者已经看见的东西。

结尾只交付当前文章已经承诺的判断、材料或下一步。没有真实经历和材料承接的协作流程、产品边界或方法论总结，不因为“文章需要收束”而硬加；它们会把一篇已成立的实践文带离主线。

## 五、证据与经验接力

- **一手信息源是事实基石**：优先官方公告、原始论文、产品文档、当事人原文和真实产品证据。
- **二手聚合源是线索与写法参考**：用于发现背景、争议和表达方式；关键事实尽量回溯一手来源。
- **碎片承载既有判断，不自动等于事实成立**：涉及外部事实时仍要回溯原始来源。
- 论点缺外部事实时进入 `507-research`，缺可复用观点或经验时进入 `507-mine`，替代解释未比较时进入 `507-fuse`；新证据动摇写作合同时，暂停成文并经 `507-grill` 重选用户拥有的路径后返回。

发现可复用的写作失败时，先搜索项目 `doc/经验笔记.md`；文件不存在则按项目约定惰性创建。没有同类条目就按“现象 + 做法 + 证据”记录，已有条目只追加独立作品证据。跨至少三篇作品重复、且能解释共同机制时，再主动提出提升为主规则；单次事故不直接改本文件。

## 六、交付门

交付前完整通读，只检查五件事；未通过就修正文，不用清单宣言代替修复：

1. **写作合同一致**：正文仍在回答当前中心判断，论证链没有漂移，结尾没有另起主线。
2. **证据承重**：关键事实有一手来源或明确降格；作者判断没有伪装成事实或成熟概念的原创命名；每个核心争点的替代解释、反例和边界均已在创意区比较并被有意识地采用或舍弃；个案或情绪不能独自承载对群体、行业或制度的概括——要么补足横向证据，要么收窄为个案解释。
3. **全文成立**：每一部分都在新增事实、因果或必要判断；删掉不影响全文的空心句直接删。
4. **读者入口兑现**：开头让读者看见这和自己的什么处境有关；若承诺提示词、模板等资源，正文在约定位置交付完整内容，而不是用空洞钩子吊着读者。相关性来自真实处境，不把情绪或个案伪装成证据。
5. **交付格式正确**：遵循目标平台与作品区约定。中文正文进入发布收口时，普通正文段落统一加两个全角空格 `　　`；标题、列表、引用、图片、表格和代码不加。

完成标准是经过共创后可发布的主稿，不是把“声音层需要用户自行手改”当作提前停工的理由。

## 完成与接力

- **完成信号**：唯一主稿通过写作合同、证据承重、全文成立、读者承诺和目标平台格式检查，已达到可发布状态。
- **产物**：作品区中的可发布书面主稿，以及同步更新的创意材料与实际附件索引。
- **候选出口**：外部证据、碎片、竞争解释或用户取舍不足时分别进入 `507-research`、`507-mine`、`507-fuse`、`507-grill` 后返回；需要现场讲述时进入 `507-stage`；需要小红书图片文章时进入 `507-rednote`；书面主稿即终点时直接结束。
- **回退条件**：中心判断、论证链或事实承重仍不成立时不进入平台适配，不用视觉包装掩盖内容缺口。
