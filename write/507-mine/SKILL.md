# 挖掘（mine）

写作工作流的**最上游**：把素材变成碎片。

解决的核心问题是 **素材来了，第一反应不该是"写什么"，而是"它围绕哪些核心争点提供了什么材料"**。外部文章人人能读，用户的想法随时会冒；如果不先拆可复用判断、不和已有判断碰撞、不把支持证据与竞争解释并排，就急着写，写出来要么是搬运、要么是先锁定结论后挑选素材。

`507-mine` 做的是：**素材 → 先看知识库 → 围绕核心争点有目的外搜多角色材料 → 产出可复用碎片 + 交接材料清单**。它不替文章聚合，也不替用户选结论。

## 覆盖什么（三种入口）

mine 吃三类素材，统一走一套流程：

1. **用户的想法/半想法/金句/模糊选题**——用户提出一个点，agent 基于它碰撞
2. **外部文章**（公众号/博客/帖子/采访/演讲）——用户提供链接或文本
3. **视频拉片内容**——`507-breakdown` 拉完片后，把拉片包交给 mine 挖内容判断

不管哪种入口，mine 的持久产物都是**碎片**（进 `01-碎片/`）；同时输出按核心争点整理的材料交接清单，供 fuse 使用。材料交接不是选题、不是正文，也不为一次性资料新建目录。

## 核心纪律（不可违反）

1. **先看知识库，再外搜**——任何素材进来，先搜 `01-碎片/`、`03-作品/` 和 `04-知识库/` 三处，看有没有同主题/相关判断。**`03-作品/` 必须查**：判断类候选尤其要先确认是不是已有作品的枢纽判断（作品正文引用的核心 claim 算重复，不止碎片）。这一步不是为了"找到最像的碎片"，而是：① 做去重（避免产重复碎片、避免和已写进作品的判断撞车）；② 让外搜有方向（基于已有判断做有目的的搜索，不是盲目搜）。
2. **默认外搜 + 按论证角色收材料**——除非用户指定了具体网站/素材（"就看这篇""只查这个项目"），否则围绕每个核心争点外搜：支持证据、反例、替代解释、适用边界/缺口、时间或制度语境。理由：同一事实可能支持不同解释；只找正反例仍会把替代解释误当作反驳，导致文章过早收敛。
3. **外搜数量下限**——支持证据或替代解释合计至少 3 篇，反例或限制材料至少 1 篇（查不到也认，但默认要找）；多则每类 5 篇上限，再多变噪音。
4. **边产边落碎片流水线**——可复用判断一旦成立（过了收录门槛），立刻走碎片流水线三步（建文件 → 维护 `00-索引/碎片图谱.canvas` → 判断知识库页生长），不攒着批量落。一次性案例或只服务当前争点的资料留在交接清单，不硬升格为碎片。理由：碎片是一等公民，落进 `01-碎片/` 后 fuse 检索时才能同时看到旧+新。
5. **过收录门槛才落盘**——一个东西要进 `01-碎片/`，至少同时满足三条：① 脱离原素材也能成立；② 未来会被第二次引用；③ 指导普遍判断，不是一次性实现细节。不满足的留在对话里，不落盘。
6. **不产选题候选，也不聚合结论**——选题候选归 `507-fuse`。mine 停在“产出碎片 + 按核心争点交接材料”：可说明材料分别承担什么论证角色，但不得决定哪种解释最强、不得给文章排论证链、不得自动产出候选 idea。
7. **不写正文**——正文归 `507-forge`。mine 不列文章大纲、不写开头、不塑形。

## 工作流程

### 1. 识别入口类型

- **想法入口**：用户提出一个点（"我觉得 X 是这样的""我有个想法"）。这是最纯粹的种子——没有外部素材，全靠碰撞。
- **文章入口**：用户提供链接或文本。先抓内容（公众号有验证码拦截时用 agent 过验证）。
- **视频拉片入口**：用户说"挖一下这个拉片包"或 breakdown 完成后顺势进 mine。读 breakdown 产出的拉片包内容。

### 2. 先看 vault（必做，外搜前）

搜 `01-碎片/`、`03-作品/` 和 `04-知识库/`，回答：
- vault 里有没有同主题碎片？（去重 + 找关联）
-用户在这个主题上已经有什么判断？（让外搜有方向，让新碎片能和旧判断互链）
- 这个素材和用户哪条既有判断能产生张力？

### 3. 有目的外搜 + 建材料角色清单（默认做，除非指定了具体素材）

先从原素材和既有碎片识别会影响后续写作的**核心争点**，再围绕每个争点做有方向的外搜：
- **支持证据**（与替代解释合计 3-5 篇）：支撑某种解释的事实、案例或研究
- **反例**（1-5 篇）：让某种解释站不稳的事实、限制条件或反面案例
- **替代解释**：同一事实可能对应的另一种机制、价值判断或当事人理解
- **适用边界/缺口**：现有材料能推出什么、不能推出什么，还缺什么
- **时间或制度语境**：变化中的规则、历史阶段和本地讨论，避免把过程中的问题写成静止本质

外搜结果按上述角色形成**材料交接清单**。它不是“佐证池对反驳池”的淘汰赛：被挑战的判断可以修正后落成碎片，也可以保留为 fuse 未来可选择的文章路径。

### 4. 拆可复用碎片，保留交接材料（核心动作）

从原素材和材料交接清单一起拆，产出一批多元碎片。每个碎片候选要过收录门槛（纪律 5）。

反例或替代解释顶翻的判断，要么修正后落（写清限制），要么不落；但相关材料仍留在交接清单中，不得因为不支持原判断而丢弃。

### 5. 边产边走碎片流水线

每个碎片成立就立刻：
1. 建 `01-碎片/{标题}.md`（按 frontmatter 约定，配来源铸定和相关碎片 `[[]]` 互链）
2. 维护 `00-索引/碎片图谱.canvas`（加节点 + 受控词表边）
3. 判断是否触发知识库页生长（某主题碎片聚拢到能独立成二次学习入口时，提示进 `507-cast`）

### 6. 交接材料，不抢下游

挖完后只交接，不聚合：
- 本次产出了哪些可复用碎片（列清单）
- 每个核心争点下有哪些支持证据、反例、替代解释、边界/缺口和时间/制度语境（附来源锚点）
- 哪些材料已足够交给 fuse 组合，哪些仍缺关键事实

材料交接可在当前对话直接传给 fuse；若之后才融合，fuse 必须从相关碎片、来源锚点和知识库重新核对。mine 不自动产候选 idea（归 fuse），不自动写正文（归 forge）。

## 和其他 skill 的边界

| skill | 分工 |
|---|---|
| **mine** | 素材 → 碎片（本 skill） |
| **fuse** | 碎片 → 候选 idea（mine 的下游，融合层） |
| **forge** | idea → 长文（fuse 的下游，成文层） |
| **breakdown** | 视频 → 拉片包（mine 的视频入口前置，不外搜只拉片） |

## 启动姿势

当用户：
- 抛一个想法/半想法/金句 → 想法入口
- 丢一个文章链接/文本 → 文章入口
- 说"挖一下这个拉片包"/breakdown 完顺势 → 视频拉片入口
- 说"帮我拆这篇文章""这个能不能写""我有个想法" → mine

你就：
1. 识别入口类型。
2. 先搜 `01-碎片/` 和 `04-知识库/` 看既有判断。
3. 默认围绕核心争点外搜多角色材料（除非指定了具体素材）。
4. 拆可复用碎片，并按论证角色整理交接材料。
5. 边产边走碎片流水线落 `01-碎片/`。
6. 交接材料给 fuse，不抢 fuse/forge。

## 红线

- 不跳过"先看知识库"直接外搜（会让外搜盲目、新碎片无法和旧判断互链）。
- 不把单篇文章/单个想法直接落成碎片而不外搜多角色材料（除非用户明确指定"就看这个"）。
- 不产选题候选（归 fuse）。
- 不写正文（归 forge）。
- 不过收录门槛的判断硬落盘（污染碎片池）。
- 不和 fuse / forge 抢活。
