# 507 挖掘（mine）

写作工作流的**最上游**：把素材变成碎片。

解决的核心问题是 **素材来了，第一反应不该是"写什么"，而是"能提炼出哪些值得长期沉淀的判断"**。外部文章人人能读，507 自己的想法随时会冒，但如果不先拆成碎片、不先和已有判断碰撞、不先用正反例压测，就急着写，写出来要么是搬运、要么是自说自话。

`507-mine` 做的是：**素材 → 先看 vault → 有目的外搜正反例 → 产出多元碎片**。

## 覆盖什么（三种入口）

mine 吃三类素材，统一走一套流程：

1. **507 自己的想法/半想法/金句/模糊选题**——507 抛一个点，agent 基于它碰撞
2. **外部文章**（公众号/博客/帖子/采访/演讲）——507 丢链接或文本
3. **视频拉片内容**——`507-breakdown` 拉完片后，把拉片包交给 mine 挖内容判断

不管哪种入口，mine 的产物都是**碎片**（进 `01-碎片/`），不是选题、不是正文。

## 核心纪律（不可违反）

1. **先看 vault，再外搜**——任何素材进来，先搜 `01-碎片/`、`03-作品/` 和 `04-知识库/` 三处，看有没有同主题/相关判断。**`03-作品/` 必须查**：判断类候选尤其要先确认是不是已有作品的枢纽判断（作品正文引用的核心 claim 算重复，不止碎片）。这一步不是为了"找到最像的碎片"，而是：① 做去重（避免产重复碎片、避免和已写进作品的判断撞车）；② 让外搜有方向（基于已有判断做有目的的搜索，不是盲目搜）。
2. **默认外搜 + 收正反例**——除非 507 指定了具体网站/素材（"就看这篇""只查这个项目"），否则默认要外搜类似观点 + 反例。理由：任何观点都有正反两面，正反例让碎片/判断立体起来，否则是单方面复述。反例那篇往往本身就是强判断，能直接顶翻原素材里站不稳的论点。
3. **外搜数量下限**——类似观点至少 3 篇，反例至少 1 篇（查不到反例也认，但默认要找）。多则各 5 篇上限，再多变噪音。
4. **边产边落碎片流水线**——碎片一旦成立（过了收录门槛），立刻走碎片流水线三步（建文件 → 维护 `00-索引/碎片图谱.canvas` → 判断知识库页生长），不攒着批量落。理由：碎片是一等公民，落进 `01-碎片/` 后 fuse 检索时才能同时看到旧+新。
5. **过收录门槛才落盘**——一个东西要进 `01-碎片/`，至少同时满足三条：① 脱离原素材也能成立；② 未来会被第二次引用；③ 指导普遍判断，不是一次性实现细节。不满足的留在对话里，不落盘。
6. **不产选题候选**——选题候选归 `507-fuse`。mine 停在"产出碎片"。如果挖掘过程中发现某几条碎片明显能融成一个选题，只口头提示"这几条值得进 fuse 看看"，不自动产出候选 idea。
7. **不写正文**——正文归 `507-forge`。mine 不列文章大纲、不写开头、不塑形。

## 工作流程

### 1. 识别入口类型

- **想法入口**：507 抛一个点（"我觉得 X 是这样的""我有个想法"）。这是最纯粹的种子——没有外部素材，全靠碰撞。
- **文章入口**：507 丢链接或文本。先抓内容（公众号有验证码拦截时用 agent 过验证）。
- **视频拉片入口**：507 说"挖一下这个拉片包"或 breakdown 完成后顺势进 mine。读 breakdown 产出的拉片包内容。

### 2. 先看 vault（必做，外搜前）

搜 `01-碎片/`、`03-作品/` 和 `04-知识库/`，回答：
- vault 里有没有同主题碎片？（去重 + 找关联）
- 507 在这个主题上已经有什么判断？（让外搜有方向，让新碎片能和旧判断互链）
- 这个素材和 507 哪条既有判断能产生张力？

### 3. 有目的外搜 + 收正反例（默认做，除非指定了具体素材）

基于上一步看到的碎片，做有方向的外搜：
- **类似观点**（3-5 篇）：佐证、延伸、不同角度的同类判断
- **反例**（1-5 篇）：反驳、限制条件、反面案例

外搜结果作为**佐证池/反驳池**，第 4 步拆素材时对照——经得起反例的判断才落碎片，被顶翻的判断要么修正要么不落。

### 4. 拆碎片（核心动作）

从原素材 + 佐证池 + 反驳池一起拆，产出一批多元碎片。每个碎片候选要过收录门槛（纪律 5）。

拆的时候对照反例池：被反例顶翻的判断，要么修正后落（标注反例限制）、要么不落。

### 5. 边产边走碎片流水线

每个碎片成立就立刻：
1. 建 `01-碎片/{标题}.md`（按 frontmatter 约定，配来源铸定和相关碎片 `[[]]` 互链）
2. 维护 `00-索引/碎片图谱.canvas`（加节点 + 受控词表边）
3. 判断是否触发知识库页生长（某主题碎片聚拢到能独立成二次学习入口时，提示进 `507-cast`）

### 6. 收尾提示，不抢下游

挖完后只做提示：
- 本次产出了哪些碎片（列清单）
- 哪几条碎片明显能融成选题 → 建议"值得进 fuse 看看"
- 反例池里有没有值得单独成碎片的强判断 → 单独提示

不自动产候选 idea（归 fuse），不自动写正文（归 forge）。

## 和其他 skill 的边界

| skill | 分工 |
|---|---|
| **mine** | 素材 → 碎片（本 skill） |
| **fuse** | 碎片 → 候选 idea（mine 的下游，融合层） |
| **forge** | idea → 长文（fuse 的下游，成文层） |
| **breakdown** | 视频 → 拉片包（mine 的视频入口前置，不外搜只拉片） |

## 启动姿势

当 507：
- 抛一个想法/半想法/金句 → 想法入口
- 丢一个文章链接/文本 → 文章入口
- 说"挖一下这个拉片包"/breakdown 完顺势 → 视频拉片入口
- 说"帮我拆这篇文章""这个能不能写""我有个想法" → mine

你就：
1. 识别入口类型。
2. 先搜 `01-碎片/` 和 `04-知识库/` 看既有判断。
3. 默认外搜类似 + 反例（除非指定了具体素材）。
4. 拆碎片，对照反例池筛选。
5. 边产边走碎片流水线落 `01-碎片/`。
6. 收尾提示，不抢 fuse/forge。

## 红线

- 不跳过"先看 vault"直接外搜（会让外搜盲目、新碎片无法和旧判断互链）。
- 不把单篇文章/单个想法直接落成碎片而不外搜正反例（除非 507 明确指定"就看这个"）。
- 不产选题候选（归 fuse）。
- 不写正文（归 forge）。
- 不过收录门槛的判断硬落盘（污染碎片池）。
- 不和 fuse / forge 抢活。
