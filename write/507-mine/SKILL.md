# 碎片挖掘（mine）

把素材里的可复用判断、经验、问题和证据关系挖出来，沉淀为碎片。

`507-mine` 回答的是：

> 这份素材里，哪些东西值得进入我们的长期碎片池？

它不负责系统查清外部世界；那是 `507-research`。两者可以独立使用，也可以组成 `research → mine`：先得到可靠证据，再把证据转成自己的碎片与观点。

## 输入

可接受四类素材：

1. 用户的想法、半想法、经验、金句或问题；
2. 用户指定的文章、访谈、笔记或既有文本；
3. `507-research` 的证据交接；
4. `507-breakdown` 的 `video_completed` 拉片包。

输入可以只有一个观点，不要求先有完整选题。

## 输出

- 可独立复用的碎片，写入知识库既有 `01-碎片/`；
- 必要的来源锚点、证据状态、反例或适用边界；
- 本轮材料交接：哪些碎片可以继续融合，哪些仍缺证据；
- 按既有规则维护碎片图谱和相关互链。

## 核心纪律

1. **先查既有资产**：先搜 `01-碎片/`、`03-作品/` 与 `04-知识库/`，避免重复沉淀，并找到应互链或更新的既有判断。
2. **挖的是可复用单元**：一个碎片承载一个能脱离本次素材继续被理解和组合的判断、经验、问题或证据关系；原文摘要不是碎片。
3. **观点允许有不同证据状态**：事实主张要有来源；个人经验、假设和待验证观点可以沉淀，但必须明确身份与边界，不能伪装成已证实事实。
4. **不内置系统调研**：读取用户指定素材和既有知识库属于 mine；需要跨来源核实、时效查证、反例搜索或机制研究时，调用 `507-research`，消费证据交接后再继续挖掘。
5. **反例可以改写碎片**：证据推翻原判断时，修正范围、降级为问题或不落盘；不为了保住一句漂亮话丢弃反证。
6. **不抢下游**：不组合成候选 idea，不替用户选择写作方向，不写正文。

## 碎片门槛

至少满足一项：

- 提供新的判断、经验、问题或反常识关系；
- 为既有判断补充关键证据、反例、边界或时间语境；
- 能与既有碎片形成有意义的支持、冲突、因果或延伸关系。

纯摘抄、换句话说、已有作品中已充分承载的重复判断，不新建碎片；优先补充既有条目或只留来源锚点。

## 工作流程

1. 识别素材类型与用户这次想保留的东西。
2. 检索既有碎片、作品和知识库，决定新建、更新、互链或跳过。
3. 拆出候选碎片，标注它是事实、经验、观点、假设还是问题。
4. 检查证据是否足以承载表述；不足且影响成立时，转 `507-research` 补证后返回。
5. 逐条落盘并维护来源、边界、互链与碎片图谱。
6. 输出材料交接，不替 `507-fuse` 或 `507-cast` 聚合。

## 材料交接

```md
## 本轮碎片
- <碎片标题> — <类型 / 证据状态 / 关键边界>

## 关系
- <支持、冲突、因果、延伸或应更新的既有条目>

## 可继续使用
- <适合交给 fuse、cast 或后续写作的材料>

## 证据缺口
- <仍需 research 的事实问题；没有则省略>
```

## 完成与接力

- **完成信号**：候选素材已完成去重、证据身份判断和碎片化；该落盘的碎片已落盘并互链。
- **产物**：碎片文件、图谱/互链更新，以及对话中的材料交接。
- **候选出口**：碎片需要组合成候选 idea 时进入 `507-fuse`；碎片簇足以形成主题页时进入 `507-cast`；外部证据不足时进入 `507-research` 并在补证后返回；用户只要求记录观点时直接结束。
- **回退条件**：素材尚不能形成可复用单元时，保留为待验证问题或不落盘，不硬造碎片。

## 红线

- 不把系统外搜、竞品比较或机制核实塞回 mine；
- 不把单篇摘要直接当自己的碎片；
- 不把观点、经验或假设写成已核实事实；
- 不产候选 idea，不写正文，不替用户决定最终表达方向。
