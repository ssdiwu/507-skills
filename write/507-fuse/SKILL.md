# 507 融合（fuse）

写作工作流的**中间层**：把碎片融合成候选 idea。

解决的核心问题是 **有了碎片，但碎片是散的判断；要写成一篇，需要一个"骨架"——用哪条碎片作主体、哪些作支撑、哪些没用上留着**。fuse 做的就是这个融合判断，但产物是**候选**（交 507 审），不是定稿。

`507-fuse` 做的是：**碎片 → 检索挑骨架 → 产候选 idea 交审**。

## 覆盖什么

- mine 产出了一批碎片（本次新落 + 既有），507 想看"这些能不能成题"
- 507 直接抛几个已有碎片，问"这几个组合起来够不够写一篇"
- 定期回看 `01-碎片/`，发现某主题碎片聚拢到够成题，主动提示融合机会

产物是**候选 idea**，落 `02-创意/候选/`，交 507 审"哪些融合得好、哪些勉强"。

## 核心纪律（不可违反）

1. **检索是必做前置**——融合前必须先翻看 `01-碎片/`（本次新碎片 + 既有碎片）。理由：每个候选 idea 都默认要检查"能否融合既有碎片"——除非确实没有合适的，否则必须融合。不允许"新 idea 完全不包含已有碎片"。
2. **融合是首选，不硬凑是底线**——有合适的既有碎片就融合；确实没有也不硬嫁接（硬嫁接会稀释判断）。这呼应 `[[三次原则]]`、`[[不是为了好而好]]`。
3. **挑骨架用时效性/主体性**——一批碎片产出后，不是全部都用。挑有时效性（最近刚提出、有当下钩子）或主体性（最强、最能承重）的碎片作骨架，其他作支撑；没用上的留在 `01-碎片/` 供以后。本次产出不一定本次全用。
4. **产候选，不拍板**——融合结果产出**候选 idea**进 `02-创意/候选/`，交 507 审查"融合质量、观点是否立得住"。agent 不替 507 决定哪个 idea 成立。
5. **不是每个碎片都要立刻进 fuse**——只有"几个碎片聚拢到够格写一篇"时才进 fuse。拆出来但还不够格的碎片，先沉淀在 `01-碎片/`，等下次有新碎片再融合。不"拆一个碎片就硬造一个 idea"。
6. **不写正文**——正文归 forge。fuse 只产候选 idea（选题卡形态），不列文章大纲、不写开头。

## 工作流程

### 1. 检索碎片（必做）

- 本次 mine 新落的碎片（查 `01-碎片/` 最近变更）
- 既有碎片（按主题/标签搜 `01-碎片/` 和 `00-索引/碎片库.base`）
- 既有候选 idea（查 `02-创意/候选/` 做去重，避免重复产题）

### 2. 挑骨架

从检索到的碎片里，挑作骨架的：
- **时效性**：最近刚提出的判断、有当下事件钩子的
- **主体性**：最强的、最能承重的主判断
- **张力**：几条碎片之间能形成对立/递进/补充关系的

骨架碎片 = idea 的主干；支撑碎片 = 丰富论证；没用上的 = 留 `01-碎片/` 供以后。

候选方向较多、融合质量拿不准或需要判断“现在值不值得写”时，按需读取 `references/选题判断.md`；不要把其中的辅助检查变成每次必跑的打分仪式。

### 3. 融合成候选 idea

按 `02-创意/README.md` 的选题卡纪律，产候选 idea：
- **题目**：可独立成立的选题
- **主线 claim**：一句话讲清这篇要讲什么判断
- **骨架碎片**：作主干的 1-3 条（`[[]]` 引用）
- **支撑碎片**：作论证的若干条
- **钩子**：具体场景/冲突/反常识
- **时效/主体性说明**：为什么这条碎片当下作骨架

不确定融合质量的，照样产出候选，交给 507 审——agent 在候选里标注"这条融合我拿不准，X 和 Y 的关联可能勉强"。

### 4. 落 `02-创意/候选/` 交审

候选 idea 落进 `02-创意/候选/`（不是直接进 `02-创意/` 正式区），等 507 审查：
- 哪些融合得好 → 提升进 `02-创意/` 正式区
- 哪些观点勉强 → 退回调整或放弃
- 哪些碎片组合需要重新挑骨架 → 重融

### 5. 维护选题图谱

每个候选 idea 落盘后，同步维护 `00-索引/选题图谱.canvas`（加节点 + 受控词表边），按 `02-创意/README.md` 的选题流水线纪律。

### 6. 不抢下游

fuse 产出候选 idea 后停。507 审通过、idea 进正式区后，下一步走 forge（长文）/ stage（讲稿）/ frame（方案）——**由 507 基于想法决定走哪个**，fuse 不替 507 分流。

## 和其他 skill 的边界

| skill | 分工 |
|---|---|
| **mine** | 素材 → 碎片（fuse 的上游） |
| **fuse** | 碎片 → 候选 idea（本 skill） |
| **forge** | idea → 长文（fuse 的下游） |
| **cast** | 碎片 → 知识库主题页（和 fuse 平行，都从碎片聚合，产物不同） |
| **stage** | idea → 讲稿（fuse 下游，允许更多 idea+碎片组合） |

**fuse vs cast 的关键区别**：都从碎片聚合，但产物不同——fuse 聚合成"值得写一篇的判断"（候选 idea，进 `02-创意/候选/`），cast 聚合成"一个主题的系统化主题页"（进 `04-知识库/`）。fuse 验收"够不够写一篇"，cast 验收"够不够成一个二次学习入口"。

## 启动姿势

当 507：
- mine 挖完一批碎片后说"看看能不能成题"
- 抛几个已有碎片问"这几个组合够不够写"
- 说"融一下""组个选题""挑骨架"

你就：
1. 检索 `01-碎片/`（本次新 + 既有）+ `02-创意/候选/`（去重）。
2. 挑时效性/主体性碎片作骨架。
3. 融合成候选 idea，不确定的标注交审。
4. 落 `02-创意/候选/` + 维护选题图谱。
5. 停在候选，不抢 forge/stage/frame。

## 红线

- 不跳过检索直接产 idea（会漏掉既有碎片，违背"默认融合"）。
- 不硬凑融合（没有合适碎片就独立，不嫁接）。
- 不把本次碎片全塞进一个 idea（挑骨架，没用上的留着）。
- 不替 507 拍板哪个 idea 成立（产候选交审）。
- 不写正文（归 forge）。
- 不和 mine / forge / cast 抢活。
