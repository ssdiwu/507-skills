# 小红书 Copy Spec

`copy-spec.md` 是逻辑文案的唯一真值。长文渲染阶段只自动分页，不擅自改字。

## 改编合同模板

```markdown
# Copy Spec

## 改编合同

- 来源主稿：`../raw.md`
- 中心判断：一句话写清原文最终要成立什么。
- 论证链：A → B → C → 结论。
- 不可删：承重事实、关键转折、结论；主稿图片清单与原始顺序。
- 图片处理：逐张记录“正文保留 / 改作封面 / 明确删除及理由”；默认全部保留。
- 可压缩：背景、重复例子、旁支说明。
- 禁止新增：原文没有的教程、清单或行动建议。
```

## 拆页纪律

1. 先按主稿章节和论证链分段，再考虑物理页数；不要按“凑够 14 页”切。
2. 默认是自动跨页长文：Copy Spec 记录章节和段落顺序，不预先承诺最终物理页数。
3. 浏览器从页头连续填到安全区底部，只在下一个完整内容块放不下时换页；章节标题与第一段绑定，但能接在上一章节后面时不另起一页。
4. “一页一观点”只用于清单、流程、对比等卡片型内容，不作为文章默认。
5. 压缩先删支撑层，不删决定文章为什么成立的转折。
6. 每个逻辑段必须写“映射”；实际物理页映射由脚本写入 `render-manifest.json`，连续页面无法映射回主稿就暂停返工。
7. 结尾必须回到主稿结论，不另起号召、教程或安全清单。

## 页数建议

- 6–9 页：短判断、单一案例。
- 10–15 页：完整文章改编。
- 超过 15 页：优先删支撑层；确有多个独立主题时拆成多篇。

页数只是密度参考，不是验收条件。验收看论证是否完整。
