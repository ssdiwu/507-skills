# 小红书图卡验收

## 先看联系表

联系表用于检查整组图，而不是检查单页细节：

- 封面能否在一秒内看懂主题。
- 页面密度是否忽高忽低。
- 图片页、判断页、过渡页是否形成节奏。
- 全篇是否保持同一主样式，没有逐页随机换皮。
- 长文是否只在章节起点出现标题，续页有没有被多余标题切断。
- 除章节收尾外，正文是否从页头填到接近页脚，避免为了多页而留下大片无意义空白。
- 后半段是否仍沿着同一条论证链推进。
- 结尾是否回到中心判断。

## 再抽查原图

至少检查封面、最密的一页、每张证据图所在页和结尾：

- 1500×2000，3:4。
- 没有文字裁切、遮挡和超出安全区。
- 正文字号在手机上可读，没有为塞字过度缩小。
- 截图没有账号、群名、项目名或其他隐私泄漏。
- 主稿图片是否逐张有去向，数量和顺序是否与改编合同一致；不得在渲染阶段静默漏图。
- 图片没有被错误拉伸；关键 UI 没被裁掉。若要删图，必须回到改编合同逐张说明理由。
- 页码连续，封面是否编号与 Copy Spec 一致。

## 内容回归

- 每页能映射回 `copy-spec.md`。
- 逐字文案没有在 HTML 或图片阶段被改写。
- 新增视觉标签只帮助理解，不新增原文没有的观点。
- 若某一页单独看合理、放回整组却不推进结论，仍判失败。

视觉好不好最终需要人看；尺寸、页数、文件完整性和溢出由脚本先挡住。
