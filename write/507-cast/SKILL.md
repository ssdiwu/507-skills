# 507 铸造（cast）

写作工作流的**沉淀层**：把碎片聚合成的主题，浇铸成系统化的知识库主题页。

和 fuse 平行——都从碎片聚合，但产物不同：fuse 聚合成"值得写一篇的判断"（候选 idea，进 `02-创意/候选/`），cast 聚合成"一个主题的系统化主题页"（进 `04-知识库/`）。

解决的核心问题是 **碎片越积越多，但没有"二次学习入口"——想系统回顾某主题时，要么在碎片图谱里漫游，要么从头重看原始素材。知识库主题页就是碎片图谱成熟簇的聚合视图，让人/AI 一进页面就能看清这个主题的全貌**。

`507-cast` 做的是：**碎片簇 → 讨论重组 → 知识库主题页**。

## 覆盖什么

- 某主题的碎片在 `01-碎片/` 聚拢到能独立成"二次学习入口"（参考下限 ≥3，不机械计数），507 想把它系统化沉淀
- mine 挖碎片时发现某主题够成知识库页，提示进 cast
- 定期回看 `01-碎片/` 和 `00-索引/碎片图谱.canvas`，发现成熟簇主动提示铸造机会

产物是**知识库主题页**，落 `04-知识库/`。

## 核心纪律（不可违反）

1. **只聚合，不融合正文**——知识库页是碎片图谱成熟簇的聚合视图（`[[索引页是人类目录不是扩展检索引擎]]`）。它聚合链接 + 元信息，不把碎片正文重新融合成一篇新长文（那是 forge 的活）。页面形态是"主题入口 + 碎片链接 + 简短定位"，不是"系统教科书"。
2. **讨论重组，不是脚本聚合**——cast 的核心是"和 507 讨论出这个主题该怎么组织"。主题页的结构（分几个子主题、哪些碎片归哪、主线是什么）靠和 507 碰撞确定，不是 agent 自己拍板。
3. **基于碎片，不凭空长**——主题页的每一段都要能回指到 `01-碎片/` 里的碎片。没有碎片支撑的主题/子主题，不写进知识库页（先去 mine 补碎片）。
4. **触发有门槛，不滥铸**——不是几个碎片就铸。某主题碎片聚拢到能独立成"二次学习入口"才铸（参考下限 ≥3，但看语义完整度不机械计数）。不够格的继续在碎片图谱里生长，等成熟。
5. **维护知识图谱**——主题页落盘后，同步维护 `00-索引/知识图谱.canvas`（加节点 + 关系边），让知识库导航图反映新页。
6. **碎片库变动要同步**——碎片库发生新增/合并/降级/删除后，cast 要回看 `04-知识库/` 各页是否滞后：该补入新碎片的补入，被吸收/删除的碎片从页里移除（呼应 AGENTS.md"碎片库变了知识库不更新 = 聚合视图失真"）。

## 工作流程

### 1. 确认主题簇成熟度

检索 `01-碎片/` + `00-索引/碎片图谱.canvas`，看某主题的碎片是否聚拢到够成知识库页：
- 碎片数量（参考下限 ≥3，但不机械计数）
- 语义完整度（这些碎片加起来，能不能讲清一个主题的全貌，还是只有零星判断）
- 有没有现成的知识库页（去重，避免重复铸）

不够成熟的，继续在碎片图谱里生长，不强行铸。

### 2. 和 507 讨论主题结构

这是 cast 的核心动作——不是 agent 自己聚合，而是和 507 碰撞：
- 这个主题该分几个子主题/角度？
- 哪些碎片是主干，哪些是支撑？
- 主线是什么（这个主题页想让人/AI 进来看清什么）？
- 有没有缺的碎片（缺的回 mine 补）？

讨论中确定的术语当场沉淀，结构当场定。

### 3. 浇铸主题页

按 `04-知识库/` 既有页的形态，铸造主题页：
- **主题定位**：一两句话讲清这个主题是什么、为什么值得有个页
- **子主题结构**：按讨论确定的结构分节
- **碎片链接**：每节聚合相关碎片的 `[[]]` 链接 + 一句话定位（不贴正文）
- **元信息**：覆盖了哪些碎片、最后更新时间、相关主题页链接

### 4. 落 `04-知识库/` + 维护知识图谱

主题页落 `04-知识库/{主题}.md`，同步：
- 维护 `00-索引/知识图谱.canvas`（加节点 + 关系边）
- 更新 `04-知识库/README.md`（如果有的话）反映新页

### 5. 不抢下游

cast 产出知识库主题页后停。主题页是"沉淀/学习入口"，不是"要写出来的文章"——如果 507 之后想基于这个主题写长文，那是另起 fuse（从主题页的碎片里挑骨架成 idea）→ forge。

## 和其他 skill 的边界

| skill | 分工 |
|---|---|
| **mine** | 素材 → 碎片（cast 的输入来源） |
| **fuse** | 碎片 → 候选 idea（和 cast 平行，都从碎片聚合，产物不同） |
| **cast** | 碎片簇 → 知识库主题页（本 skill） |

**cast vs fuse 的关键区别**：都从碎片聚合。fuse 聚合成“值得写一篇的判断”（候选 idea），验收“够不够写一篇”；cast 聚合成“一个主题的系统化主题页”，验收“够不够成一个二次学习入口”。fuse 走向“写出来”，cast 走向“沉淀下来”。

## 启动姿势

当 507：
- mine 挖完碎片后说"这几个够成知识库页了"
- 说"把这个主题沉淀一下""铸个主题页"
- 定期回看时 agent 发现成熟簇主动提示

你就：
1. 检索 `01-碎片/` + 碎片图谱，确认主题簇成熟度。
2. 和 507 讨论主题结构（子主题、主干碎片、主线）。
3. 浇铸主题页（聚合链接 + 元信息，不融合正文）。
4. 落 `04-知识库/` + 维护知识图谱。
5. 停在沉淀，不抢 fuse/forge。

## 红线

- 不把知识库页写成长文/教科书（只聚合链接+元信息，融合正文归 forge）。
- 不脚本聚合、不和 507 讨论就自己拍结构。
- 不凭空写没有碎片支撑的主题段（缺碎片回 mine 补）。
- 不滥铸（不够成熟的簇继续生长，不强行成页）。
- 碎片库变动后不更新知识库页（会让聚合视图失真）。
- 不和 fuse / mine 抢活。
