# Changelog

本项目遵循 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/) 的基本格式；版本以 Git tag（Git 标签）为准。

## [Unreleased]

### Changed

- `507-research` 改为通用外部证据入口，默认交付可追溯证据包；`507-mine` 收缩为碎片与观点生产，证据不足时消费 research 结果后继续，不再内置系统外搜。
- `507-prototype` 前移为证据层并增加成本收益闸门、verdict 与清理状态；`507-grill` 明确可先运行测试或最小验证查清事实，`507-prd` 的测试接缝改由 agent 自行设计并告知用户。
- `507-issue` 增加 `draft → investigation → ready` 生命周期，允许同一远程 issue 先调查、通过评论积累证据，再原位升级为可领取执行合同；所有远程状态继续强制核验标签和负责人。
- `507-frame` 收缩为消费已确认需求的面向人方案层；用户取舍统一返回 `507-grill`，具体软件产品规格归 `507-prd`。
- 全部 skill 新增完成信号、产物、候选出口与回退条件；Agent 依据当前目标选择并解释接力，不再让用户选择内部 skill 名称。
- 根、common、code 与 write README 重新定义横向层、前置证据层、代码工单生命周期和三类写作入口；公开 AGENTS 模板同步采用同一路由纪律。
- 公开 README、全局 AGENTS 模板、术语与共享技能说明改用宿主中立口径；安装目录、递归发现、重载方式和显式调用语法改由目标宿主文档定义。
- `507-rednote` 的单张配图出口改为当前环境可用的配图或图像生成能力，内置样式不再暴露本地外部项目名称，并修正默认样式 ID。
- `507-breakdown` 移除固定 ASR 虚拟环境，允许通过参数或环境变量选择 Python，并补齐依赖检查与公开运行条件。
- `507-breakdown` 以中性合成内容替换外部项目语义 fixture，并明确区分静态契约样本与真实 API 证据。
- `507-breakdown` 明确分别接力内容挖掘与视频手法重组；`507-remix` 补齐输入契约、原创边界、输出、验收和视频执行出口；`507-forge`、`507-fuse`、`507-cast`、`507-stage` 与 `507-rednote` 统一把用户取舍交给 grill。
- 原 `code/507-explain` 重构为通用只读 `507-explore`：面对项目未知时读取既有共识与现实证据，在会话中累积呈现已知、未知、`frontier（当前可推进边界）`与证据边界，找到下一项明确动作后路由，不创建持久探索地图。
- 原 `507-teach` 重命名为 `507-explain`，继续聚焦单个概念理解，并在流程、状态、层级、映射或多方关系难以用纯文字讲清时选用表格、`Mermaid（图表语法）`或 `ASCII（字符图）`示意。
- `507-forge` 新增真实因果链、图片证据位置与结尾范围纪律，避免实践文章由工具开场、图片失去证明作用或在收束时扩写未承诺的方法论。
- 全局 `AGENTS.md` 示例新增最小实现梯子，要求在理解完整链路后优先复用已有实现、标准库、平台原生能力与已安装依赖，同时保留安全、错误处理、无障碍和分层验证边界。
- `507-prd`、`507-issue` 与 `507-research` 新增产物级局部自检，在交给下游前直接修正占位符、内部矛盾、范围失控、歧义、不可验证表述与证据缺口。
- `507-issue` 要求每个远程 issue 按仓库现有规范设置并核验合适的标签与负责人；只有仓库权限不足时才允许元数据未落地，且必须明确报告待补内容与未完成状态。

### Breaking

- 显式项目探索调用从 `507-explain` 迁移到 `507-explore`；原 `507-teach` 的概念解释调用迁移到新的 `507-explain`。不保留重复 skill 别名，旧显式名称会失效或改变语义。

## [0.2.0] - 2026-07-16

### Changed

- `code/` 与 `common/` 的公开 skill 名称改为功能直译并继续保留 `507-` 命名空间：`setup`、`prd`、`issue`、`prototype`、`inspect`、`explain`、`fix`、`map`、`review`、`commit`、`simplify`、`test`、`tdd` 与 `handoff`；`grill`、`research` 和 `teach` 保持不变。
- 非写作技能的标题、流程图、路由说明、全局 `AGENTS.md` 模板及 Pi/Codex 本机配置同步使用功能语言；`write/` 保留既有创作命名体系。

### Removed

- 移除 `code/` 与 `common/` 的旧施工名称目录，不保留重复 skill 别名。

### Breaking

- 显式调用需从 `507-ground`、`507-blueprint`、`507-workorder`、`507-mockup`、`507-inspection`、`507-survey`、`507-repair`、`507-asbuilt`、`507-audit`、`507-closeout`、`507-retrofit`、`507-probe`、`507-formwork` 与 `507-handover` 分别迁移到 `507-setup`、`507-prd`、`507-issue`、`507-prototype`、`507-inspect`、`507-explain`、`507-fix`、`507-map`、`507-review`、`507-commit`、`507-simplify`、`507-test`、`507-tdd` 与 `507-handoff`。

## [0.1.0] - 2026-07-16

### Added

- 新增九项跨平台代码工作流：`507-survey`、`507-repair`、`507-asbuilt`、`507-audit`、`507-closeout`、`507-retrofit`、`507-probe`、`507-formwork` 与 `507-handover`，覆盖项目勘察、根因修复、实况地图、交付审查、Git 收尾、行为不变改造、测试探测、测试先行和会话移交。

### Changed

- 本项目现在以 Pi 与 Codex 当前均支持的 `~/.agents/skills` 用户级目录共享宿主中立技能；两端保持触发边界、产物与验收一致，具体调度由各自 `AGENTS.md` 决定。
- `507-inspection` 改为输出逐卡带代码证据、置信等级和设计约束的完整报告，并交给 `507-retrofit` 做第二层验证、施工或带证据关闭；`507-ground` 收缩为项目地基巡检并把地图维护交给 `507-asbuilt`。
- `507-grill` 改为按依赖层推进：当前已知且互不影响的用户决策可批量收集，存在依赖的分支仍逐题问到底。
- `507-rednote` 增加资源分发策略：提示词、模板、清单和附件必须明确选择图卡全文、评论区钩子、系列拆分或外链附件，评论区钩子需用户明确授权。
- `507-rednote` 为保真清单增加 `M` 编号到 `sourceMap` 的逐项映射，并以渲染状态、规格 SHA-256 和产物清单作为交付硬门；封面映射同步进入渲染清单。
- `507-forge` 增加读者入口与承诺检查，并明确项目约定的唯一正文真相源，避免平台稿演变为并列主稿。
- `507-mine`、`507-fuse` 与 `507-forge` 新增核心争点与论证角色工作流：挖掘层交接支持证据、反例、替代解释、边界/缺口和时间/制度语境；融合层在存在真实竞争解释时提供 2～5 条论证组合候选；成文层缺少该材料时先回补并等待用户选择。
- `507-stage` 增加 AI 协作课编排：先建认知地图、再用可预览 Demo 落地概念；明确 `DESIGN.md`、Skill（技能）与 MCP（模型上下文协议）的层次，并要求 HTML 课件在浏览器完成视觉核验。`507-fuse` 与 `507-frame` 同步要求未收口创意/方案显式区分已确认共识、讨论稿和待定项。

### Fixed

- `507-rednote` 新增承重内容保真清单、头像识别和封面/正文联合选样检查，避免完整提示词、模板、图片或作者识别资产在图卡改编中被静默遗漏。

### Removed

- 移除 `prompts/` 下九份 Pi 专属提示词模板，不再维护与共享 skill 重复的第二事实源。

### Breaking

- `/explain`、`/fix`、`/map`、`/review`、`/commit`、`/simplify`、`/test`、`/tdd`、`/handoff` 不再提供；改用自然语言触发，或使用 Pi 的 `/skill:<name>` 与 Codex 的 `$<skill-name>` 显式调用对应 `507-*` skill。
- `507-rednote` 的 `cover` 页面现在必须提供非空 `sourceMap`；既有 `rednote-project.json` 需补充封面对应的保真编号与来源说明。

## [0.0.4] - 2026-07-14

### Fixed

- `507-breakdown` 的关键帧图片理解改用宽容 JSON 解析器，接受 MiniMax-M3 用 Markdown 围栏包裹的有效 JSON；仍严格校验描述、回显锚点、肯定可见性和非空证据，并允许证据对锚点的自然语言转述，新增无 API key（接口密钥）回归测试。

## [0.0.3] - 2026-07-14

### Changed

- 全部 skill、参考资料、fixture（测试样本）、prompt（提示词）和仓库说明改用中性角色与可替换路径；保留 `507-*` 作为稳定的 skill ID（技能标识），移除私有项目、issue 和机器路径示例。
- `507-rednote` 以 `editorial-default` 取代此前的编辑文档预设；现有项目的 `stylePreset`（样式预设）需更新为新 ID，并已同步 schema（模式）、渲染器、样本和文档。
- `507-grill` 新增“决策所有权闸门”：API（接口）设计、参数命名、内部模块、测试与工程验证默认由 agent 自行收口，只把产品意图、权责边界、不可逆成本、体验和风险取舍交给用户；已被事实或既有决策排除的路径不得凑成候选，发现理解错误时必须先修正并撤回错误设计，不得把修复责任继续转交给用户。
- `507-forge` 将“活人感”约束为可改变判断的具体上下文（亲历、证据、失败、取舍与边界）；材料不足时回到创意区或 `507-mine` 补证，不以第一人称、口癖或情绪词伪造作者声音；写作合同与交付门同步区分个案事实和总体解释，禁止用单一故事或情绪直接概括群体、行业或制度。

## [0.0.2] - 2026-07-14

### Added

- 新增 `507-rednote`：把成熟书面作品保真改编为小红书 3:4 图片文章；默认将逻辑文案按真实高度连续填页，章节标题不强制换页，主稿图片逐张保留去向与顺序，结尾自动避免孤页；接入 `RedNoteImage` 的 17 套主题并保留编辑文档风，内置样式联系表、本地 HTML 渲染、1500×2000 图卡导出、18 样式溢出检查和物理页来源映射。

### Changed

- `507-grill` 的决策提问改为默认呈现 2–5 个有真实权衡的候选并标出推荐项；禁止同向变体凑数，并要求主动检查可成立的反向路径。
- `507-forge` 从显式模式、调性分流和四层清单收敛为写作合同驱动的共创循环，支持 Agent 自动调用 `mine / fuse` 补证、创意材料回写和书面平台保真改编。
- 选题判断参考迁入 `507-fuse`，只在候选方向难以取舍时按需加载。
- 代码项目规范改为从开局建立基于 Keep a Changelog 的 `CHANGELOG.md` 和 `Unreleased` 段；`/fix`、`/commit`、`/review` 与 `507-ground` 同步检查用户可感知变更和首次初始化边界。
- 全局 `AGENTS.md` 示例不再预设使用者昵称，改由使用者偏好或项目约定决定称呼。

### Removed

- 移除 `507-forge/references/` 下的调性包、技法词表与旧选题方法，避免把协作对话口癖当成文章声音或让单次失败永久立法。

## [0.0.1] - 2026-07-13

### Added

- 公开仓库入口：安装说明、贡献指南、安全政策与全局 Agent（智能体）配置模板。

### Changed

- `507-forge`（长文锻造）新增原创中心判断与结论层级约束，区分保留口癖和机械添加口癖，并禁止把成熟概念冒充作者命名。
- `507-forge` 按项目约定识别唯一主稿路径，不再把 `raw material`（输入素材）误当作固定文件名规则或擅自建立平行稿。

历史提交可通过 Git（版本控制）记录查看；首次公开发布前不追溯整理版本条目。
