# 项目探索（`explore`）

面对一个还看不清全貌的项目问题，先建立共同上下文，再找出当前真正能够推进的未知。探索是一段有起止的临时会话，不是长期运行的编排器；长期共识继续由项目文档、术语、决策、经验、规格和工单承载。

`507-explore` 做的是：**项目未知 → 证据支持的认知边界与下一步路由**。

## 覆盖什么

- 定位一个现象、想法或问题在项目中的入口、上下游与证据范围；
- 解释当前项目机制，同时区分已经确认与仍未核实的部分；
- 面对暂时无法完整规划的方向，找出当前可解决的一个 `frontier（当前可推进边界）`；
- 在代码、写作或其他项目中，为后续专门技能建立输入上下文。

## 不覆盖什么

- 用户只是不理解一个概念或术语 → 用 `507-explain`；
- 外部参照物的机制与借鉴判断 → 用 `507-research`；
- 已经明确需要用户决定的权衡 → 用 `507-grill`；
- 系统性架构体检 → 用 `507-inspect`；
- 实现、修复、写规格、建工单或修改项目文档 → 退出本技能后进入对应工作流。

## 核心纪律

1. **先读既有共识**：遵循项目 `AGENTS.md`、根 `README.md`、`doc/README.md` 和项目规定的阅读顺序；按需读取术语表、决策档案、经验笔记、相关分层 README、代码、测试、配置或写作材料。
2. **不伪造完整路线**：只写证据已经支持的已知、尚未支持的未知，以及前置条件已经满足的当前可推进边界；未来问题仍看不清时留在未知区，不补成假计划。
3. **能查的事实立即查**：项目文件、运行输出、文档或小型只读验证可以回答的，不转问用户。外部参照物需要系统调研时路由 `507-research`。
4. **事实与决策分开**：只有产品意图、权责边界、不可逆成本、体验或风险取舍需要用户决定；一旦当前可推进边界变成这类问题，携带探索快照转入 `507-grill`。
5. **会话内累积快照**：每完成一轮有效查证或准备切换工作流时，重写最新、完整的探索快照；不追加流水账，不让旧结论和新证据并排冲突。
6. **只读且临时**：不修改代码、配置或项目文档，不创建持久探索地图。对话结束后快照可以消失；够格的长期结论由后续动作沉淀到项目既有载体。
7. **找到下一步就停止**：问题已经能够直接回答就给结论；需要专门动作就明确路由，不把 `507-explore` 扩成 `507-research`、`507-grill`、`507-prototype` 或实施总控。

## 工作流程

1. **锁定探索目标**：用一句话说明这次要弄清什么，以及暂不处理什么；用户已说清时不重复确认。
2. **读取项目依据**：从项目规则和权威文档开始，再按需进入相关实现、测试、配置或材料；没有明确路径时先做窄范围搜索。
3. **建立首个快照**：列出已知、未知、当前可推进边界与证据边界；只能从局部判断时明确标注已读范围。
4. **推进当前可推进边界**：读取证据或做只读验证；得到新事实后重新计算已知、未知和下一可推进边界。
5. **收束或路由**：直接回答已经弄清的问题；概念不懂转 `507-explain`，外部机制转 `507-research`，用户决策转 `507-grill`，需看见后判断转 `507-prototype`，已足够清楚则进入规格、工单或正常实施。

## 会话输出

探索过程中按实际复杂度呈现并累积重写：

```md
## 当前探索快照

### 目标
- <这次要弄清什么>

### 已知
- <有项目依据支持的事实 + 依据>

### 未知
- <当前证据仍不足的部分>

### 当前可推进边界
- <现在具备前置条件、最值得先解决的问题>

### 证据边界
- <已读范围、未核实范围与置信限制>
```

结束时给出：一句话结论、仍未解决的未知，以及唯一推荐的下一工作流。简单问题不机械展开全部标题。

## 与相邻技能的边界

| 技能 | 回答的问题 |
|---|---|
| `507-explore` | 项目里已经知道什么、还不知道什么、现在先查什么？ |
| `507-explain` | 这个概念到底是什么意思，怎样讲我才能懂？ |
| `507-research` | 外部对象真实怎么做，对当前项目有什么借鉴？ |
| `507-grill` | 在已知前提下，需要用户决定的权衡最终选什么？ |
| `507-prototype` | 必须看见或操作一个低成本原型后才能判断什么？ |

## 红线

- 不跳过项目文档直接把代码猜测写成事实；
- 不把未知补成完整计划；
- 不把可查事实转交用户决定；
- 不持久化探索快照或建立第二事实源；
- 不在只读探索中顺手实施、修文档或创建提交。
