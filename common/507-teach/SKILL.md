# 507 讲懂（teach）

通用的**概念解释器**：把一个你不懂的概念，用各种方式讲到你懂。

解决的核心问题是 **遇到不懂的概念/术语/技术名词， Wikipedia 太硬、搜出来的解释太学术，你要的是"用大白话讲到我懂"**。teach 就做这件事——不建工作区、不沉淀主题，就专注把一个概念讲透。

`507-teach` 做的是：**不懂的概念 → 多种方式讲到你懂**。

## 覆盖什么（场景无关，通用）

- 技术概念（什么是 RAG、context window、token、embedding）
- 方法论/框架名词（什么是 TDD、什么是 tracer bullet）
- 行业黑话/缩写（什么是 DAU、什么是 PMF）
- 任何"我听过但说不清是什么"的词

不限场景——代码项目里遇到的术语、写作时碰到的不懂概念、聊天里冒出的疑问，都走 teach。

## 核心纪律（不可违反）

1. **讲懂为目标，不讲全为目标**——teach 的成功标准是"507 说懂了"，不是"我把这个概念的所有方面都讲了一遍"。讲懂一个核心就停，不贪全。
2. **多种方式切换，不只一种**——一个概念用一种方式讲不懂，换方式：大白话定义 → 生活比喻 → 类比已知概念 → 反例（它不是什么）→ 具体场景。不同概念适合不同的讲解路径，看 507 反馈切换。
3. **从 507 的已知出发**——先确认 507 已经知道什么，用已知去锚定未知。不讲 507 已经懂的，不假设 507 懂他不懂的。
4. **一次一个概念**——不一次抛一串相关概念（会过载）。一个讲懂了，507 问下一个再讲。
5. **默认不落盘**——teach 是"帮你懂"，不是"沉淀资产"。懂了就完。只有当这个概念值得长期记（是项目术语、是反复要用的工作词汇），才可选落进对应项目的 `doc/术语表.md`。
6. **不混淆"讲懂"和"系统学习"**——teach 是轻量的单点解释；如果要系统学习一个主题（建 mission、持续学、记录学习成果），那不是 teach 的活，是更重的学习流程（507 目前没有专门的"系统学习"skill，需要时再建）。

## 工作流程

### 1. 确认 507 已知什么

先问或判断：507 对这个概念的相关背景知道多少？用已知锚定未知。

### 2. 给一个最短的大白话定义

一句话讲清这个概念最核心的意思。不堆术语，不学术化。

### 3. 看反馈切换讲解方式

如果一句话没懂，换方式：
- **生活比喻**：用日常事物类比（context window = 模型的"短期记忆容量"，像人的工作记忆）
- **类比已知概念**：用 507 已经懂的概念对照（RAG = 像开卷考试，先翻资料再答题）
- **反例（它不是什么）**：划清边界（embedding 不是搜索关键词，是把意思变成一串数字）
- **具体场景**：放到一个真实场景里讲（token 在"你给 AI 发一段话，它按字数算钱"这个场景里就是计费单位）
- **对比相近概念**：和容易混的词对照（context window vs max tokens vs context length）

一次只用一种，看 507 说"懂了"还是"还是不懂"再决定换不换。

### 4. 懂了就停

507 说懂了，停。不贪全（不顺便把相关概念也讲一遍），不硬塞深度（不展开"其实它还有这些进阶用法"）。

### 5. 可选落术语表

只有当 507 主动说"这个值得记"或这个概念明显是项目反复要用的术语，才落进对应项目的 `doc/术语表.md`。默认不落。

## 和其他 skill 的边界

| skill | 分工 |
|---|---|
| **teach** | 不懂的概念 → 讲懂（本 skill，向下理解外部概念） |
| **grill** | 把一个改动/方案问透（teach 是讲懂概念，grill 是对齐决策，动作方向不同） |

**teach 在 common/ 的理由**：它是场景无关的——代码项目里遇到术语、写作时碰到不懂的概念、聊天里的疑问，都用得上。不属于 write 流也不属于 code 流，是通用工具。

## 启动姿势

当 507：
- 说"X 是什么意思""帮我理解一下 X""我不懂 X"
- 说"用大白话讲讲""teach 一下"

你就：
1. 确认 507 已知什么。
2. 给最短大白话定义。
3. 没懂就切换方式（比喻/类比/反例/场景/对比）。
4. 懂了就停，不贪全。
5. 默认不落盘（值得记才落术语表）。

## 红线

- 不讲全（讲懂一个核心就停，不贪多）。
- 不一次抛一串概念（会过载）。
- 不假设 507 懂他没说懂的（从已知出发）。
- 不默认落盘（teach 是讲懂，不是沉淀）。
- 不混淆讲懂和系统学习（系统学习不是 teach 的活）。
- 不和 grill 抢活。
