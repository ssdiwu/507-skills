# 按动作与修改权限划分代码工作流

原 Pi prompt（提示词）不按文件形式机械合并，而按用户触发的动作、允许修改的对象和停止位置划分为公开 skill（技能）。`explain`、`fix`、`map`、`review`、`commit`、`simplify`、`test`、`tdd`、`handoff` 各自保留独立入口：项目解释、错误修复、地图文档维护、交付审查、Git 提交、行为不变简化、测试工作、测试驱动实施和临时交接不是同一个动作。相邻能力通过路由接力，不用一个总控技能吞并；其中 `507-map` 以代码为证据但只改地图文档，`507-setup` 负责更广的项目初始化与全量规范检查且不修改业务代码。
