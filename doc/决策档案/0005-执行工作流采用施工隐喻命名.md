# 执行工作流采用施工隐喻命名

九项跨平台工作流继续使用 `507-` 命名空间，并采用单个英文施工隐喻词作为公开名称：`507-survey`（项目勘察）、`507-repair`（根因修复）、`507-asbuilt`（实况地图文档）、`507-audit`（交付审查）、`507-closeout`（Git 收尾）、`507-retrofit`（行为不变的内部改造）、`507-probe`（测试探测）、`507-formwork`（测试先行施工）和 `507-handover`（会话移交）。原有 explain/fix/map/review/commit/simplify/test/tdd/handoff 继续作为 description（描述）中的触发词，不再作为技能名。
