# 重排工作流层次并采用显式出口合同

项目决定把 `common/` 定义为任意阶段可调用的横向认知与协作层，而不是进入代码或写作前必须依次通过的前置层；`507-prototype` 虽位于 `code/`，在流程上归入前置证据层。外部事实由 `507-research` 形成证据交接，写作碎片与观点由 `507-mine` 生产；所有属于用户的确认统一进入 `507-grill`，面向人的方案进入 `507-frame`，具体软件产品规格进入 `507-prd`；`507-issue` 以同一载体承接 `draft → investigation → ready`，调查证据进入评论、当前结论回写正文。

每个 skill 同时声明完成信号、产物、候选出口与回退条件。出口可以有多个、返回调用它的工作流或直接结束；由 Agent 根据当前目标和证据选择并解释，不让用户记忆或选择 skill 名称。代价是每个 skill 需要维护更明确的接力合同，收益是工作流不再依赖隐藏顺序、单一出口或用户对框架内部命名的理解。
