# 拆分项目探索与概念解释

项目决定把“探索项目未知”和“解释用户不理解的概念”保留为两个独立的通用技能：原 `code/507-explain` 扩展为只读、会话级的 `common/507-explore`，读取项目共识与现实证据，累积呈现已知、未知、当前 `frontier（可推进边界）`和证据边界，并在找到下一项明确动作后路由；原 `common/507-teach` 改名为 `common/507-explain`，专注用白话、例子、反例、对比以及必要的表格、`Mermaid（图表语法）`或 `ASCII（字符图）`把单个概念讲懂。两者都不承担长期探路状态，且不保留旧名称的重复技能别名。

这次拆分把“项目本身尚未看清”和“用户尚未理解一个概念”视为不同动作：前者需要读取项目依据并管理会话内认知边界，后者需要选择最合适的解释与示意方式。代价是 `507-explain` 的公开语义发生变化、`507-teach` 的显式调用失效，并需要同步各层工作流入口；收益是触发词、修改权限、停止位置和产物边界不再重叠，同时两项能力都能在代码与写作场景复用。
