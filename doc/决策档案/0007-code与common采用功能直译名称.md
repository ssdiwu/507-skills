# code 与 common 采用功能直译名称

> 状态：其中 `survey → explain` 与保留 `teach` 的名称映射已由 [0008-拆分项目探索与概念解释](0008-拆分项目探索与概念解释.md) 取代；功能直译、`507-` 命名空间和不保留重复别名的原则继续有效。

公开 skill（技能）名称首先是检索与调用入口。`code/` 与 `common/` 的技能名、中文标题和流程说明采用功能直译，避免使用者先记住隐喻再翻译成动作；`write/` 保留已经形成稳定写作链路的创作命名。`507-grill` 因已成为稳定的用户语言且不属于施工命名，继续保留。

继续使用 `507-` 命名空间，原子改名且不保留旧 skill 别名：`ground → setup`、`blueprint → prd`、`workorder → issue`、`mockup → prototype`、`inspection → inspect`、`survey → explain`、`repair → fix`、`asbuilt → map`、`audit → review`、`closeout → commit`、`retrofit → simplify`、`probe → test`、`formwork → tdd`、`handover → handoff`。`research` 与 `teach` 已是功能名称，保持不变。该变化作为 `v0.2.0` 发布，不改写已公开的 `v0.1.0` 标签。
