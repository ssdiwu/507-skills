# Agent Skills 术语表

本表只定义 `507-skills` 工作流中的稳定项目语言，不记录实现步骤或决策理由。

## Language

**项目地图（Project Map）**：
以当前代码和权威文档为事实依据的项目空间导航，包括根/分层 `README.md`、`doc/README.md`、目录职责、入口、上下游和边界；不包含未来架构方案或全部项目文档正文。
_Avoid_: 全量文档、架构方案

**双层证据链（Two-stage Evidence Chain）**：
`507-inspect` 先为候选提供可定位的代码观察证据与置信等级，`507-simplify` 再建立行为基线或可执行验证信号；第二层成立才修改，不成立则带证据关闭。
_Avoid_: 自动实施全部候选

**宿主中立技能（Host-neutral Skill）**：
在 Pi、Codex 等宿主中保持相同触发边界、修改权限、停止位置、产物和验收标准的 skill；具体工具与调度规则留在各宿主的 `AGENTS.md`。
_Avoid_: 双宿主技能副本、宿主 adapter

**项目探索（Project Exploration）**：
围绕一个尚未看清的项目问题，读取既有共识与现实证据，在临时会话中逐步确定已知、未知、当前可推进边界和下一工作流；不创建持久探索地图。
_Avoid_: 长期探路编排、项目解释

**探索快照（Exploration Snapshot）**：
`507-explore` 在对话中累积重写的临时认知边界，包括目标、已知、未知、当前 `frontier（可推进边界）`与证据范围；不是项目长期事实源。
_Avoid_: 探索地图、项目文档

**概念解释（Concept Explanation）**：
围绕用户不理解的一个概念，用最短定义和必要的例子、反例、对比或视觉示意建立正确心智模型；不负责定位项目未知。
_Avoid_: 项目探索、系统学习
