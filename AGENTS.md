# 507 Skills 贡献规范

## 项目定位

这是 507 个人工作流的公开 Agent Skills（智能体技能）仓库。每个 skill 都是一个可独立触发的工作流入口；其 `SKILL.md`（技能说明）是该 skill 的行为边界与使用说明的权威来源。

## 阅读顺序

修改前依次阅读：

1. 根 [`README.md`](README.md)（公开定位、安装与安全边界）；
2. 本文件；
3. 目标流的 `README.md`（`write/`、`code/` 或 `common/`）；
4. 目标 skill 的 `SKILL.md`，以及它显式引用的 `README.md`、`AGENTS.md`、`references/`。

## 修改原则

- 一个 skill 只承担一个清晰动作；新增能力优先检查是否侵占已有 skill 职责。
- 触发词、输入、输出、非目标和验证方式须彼此一致；行为或依赖变化时同步 `SKILL.md` 与上级 `README.md`。
- 没有第三次真实重复，不提前抽象。
- 脚本、fixture（测试样本）和外部 API（接口）调用必须明确运行条件；密钥只从环境变量读取。
- 不提交密钥、Cookie（会话凭据）、个人 vault（知识库）、客户数据、原始私密素材或本机绝对路径。
- `templates/`（模板）与 `prompts/`（提示词）是可借鉴的公开版本；不要把使用者个人配置直接同步回仓库。

## 验证

- 纯文档改动：检查 Markdown（标记语言）链接、目录引用和触发边界是否准确。
- 脚本改动：运行目标目录已有的最小测试，再按需扩大。
- 发布前：运行密钥扫描，检查 `git diff --check`，并确认公开说明未承诺未实现的能力。

## Git（版本控制）

使用 `Conventional Commits`（约定式提交），格式为 `type: 中文描述`。一次提交只处理一个可说明的改动；不要混入 `__pycache__/`、本地测试产物或无关格式化。
