description: 中断、换会话或转交时生成交接摘要
argument-hint: "[下一段会话的重点]"
---
结合当前会话，以及以下补充内容，生成一份**交接摘要**，直接在对话里输出，不写文件：

$@

交接摘要让新会话 / 新 agent 能接着做，不重新翻完整对话；只保留继续工作所需上下文，不复制已沉淀到正式产物的内容。它不属于默认收尾——默认到 `/commit` 结束；只有中断、换会话、转交或需要压缩上下文时才用。用户给的补充说明当作下一段重点，围绕它裁剪。

## 原则

- 直接输出，不写文件，不存 repo 或 temp。
- 不复制正式产物：PRD / 决策档案 / issue / commit / diff / 文档已存在时只引用路径或 URL。
- 脱敏 API key、密码、token 和个人 / 客户敏感数据。
- 少叙事，多写状态、决策、文件、下一步。
- 涉及代码仓库时交代 Git 状态、未提交文件和验证结果。

## 输出结构

```md
# Handoff

## 当前目标
<本轮想完成什么；给了下一段重点就优先写那个>

## 已完成
- <已落地事项，引用路径而非复制全文>

## 关键决策
- <重要取舍及理由>

## 重要文件 / 入口
- `<path>` — <为什么重要>

## Git 状态 / 未提交改动
- 仓库：<path 或“非 Git 仓库”>
- 当前状态：<干净 / 有未提交改动 / 有未跟踪文件 / 不确定>
- 相关文件：`<path>`

## 验证状态
- 已运行：<命令与结果>
- 未验证：<原因与建议验证方式>

## 未完成 / 下一步
1. <下一步 1>
2. <下一步 2>

## 建议使用的 skills / prompts
- `<skill-or-prompt>` — <何时用>

## 风险 / 注意事项
- <易踩坑、必须遵守的约束、需脱敏或验证的点>
```

## 红线

- 不编造未发生的决策。
- 不把临时交接当长期事实来源。
- 不泄露敏感信息；不输出大段代码或长文档，用路径引用。
